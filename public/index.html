研究に必要な機能を、わかりやすく

複雑な設定は後回し。まず使える UI を優先して設計しています。

📁

ファイル共有

R2 にデータを保存し、プロジェクト単位でメンバーと安全に共有。大容量のシミュレーション結果も扱えます。

🗂

プロジェクト管理

ディレクトリ階層で研究テーマを整理。チームごとの作業スペースを D1 で一元管理します。

📅

スケジュール管理

実験・解析・発表の予定をプロジェクトに紐づけ。研究の見通しをチーム全員で共有します。

🖨

3D プリント連携

3dprint.mmh-virtual.jp と連携し、設計データから造形までの流れをつなぎます。

低コストで運用できる構成

Cloudflare の Pages・D1・R2 を中心に構築。 Workers は将来の API 拡張用に用意しつつ、まずは Pages Functions で必要十分な機能を提供します。

  • Pages — フロントエンドと API
  • D1 — ユーザー・ディレクトリ管理
  • R2 — ファイルストレージ
  • Workers — 将来拡張（現在は未使用）
